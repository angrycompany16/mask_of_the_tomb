  function for rotating a rect about its center, but not that much more.

Engine improvements:
- Spawning children should ideally not need to go through the scene object,
  it would be way cleaner if we could just do that as a function of the node
  I believe the problem is that Init and OnTreeAdd would not be called in 
  this case. Example:
			
      cmd.Scene().AddChild()

- The deep nesting of constructors really needs to be fixed... Example:
		
    scene.SpawnActor("Slambox",
			slamboxactor.NewSlambox(
				tracker.NewTracker(
					graphic.NewGraphic(
						transform2D.NewTransform2D(
							nodeactor.NewNode(),
						),
					), moveSpeed, posX, posY,
				),
				slamboxactor.WithPos(posX, posY),
				slamboxactor.WithSize(width, height),
			),
			cmd,
		)

- It would be very nice if we did not need to call child methods
  each time we inherit from a different class. for example:

  func (t *Transform2D) Update(cmd *engine.Commands) {
	t.Node.Update(cmd)
	...

  It would ideally be automatic to call update on the embedded 
  Node member
- Init is not called when we spawn an actor after the scene has 
  already been initialized. It is only called once a scene is 
  spawned, which means that for newly spawned objects the Camera
  becomes a nil pointer unless we call Init manually...
- Rectify() in SlamboxEnv is currently being called on demand.
  I guess it's fine, but we really shouldn't be doing that kind
  of stuff.
- Another constructor problem: If we forget to spawn some parent
  actor with an instance of its child, everything panics. It
  would be nice if we could create component A inheriting from B,
  and then somehow make a constructor for B which makes it impossible
  to forget to add an A.
